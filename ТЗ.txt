# ТЕХНИЧЕСКОЕ ЗАДАНИЕ НА РАЗРАБОТКУ ОБРАЗОВАТЕЛЬНОЙ ИГРЫ "ЯБЛОКИ И ЁЖИК"

## 1. Общие сведения

### 1.1. Назначение игры
Образовательная игра "Яблоки и ёжик" предназначена для детей дошкольного и младшего школьного возраста с целью развития навыков счета в увлекательной форме через интерактивное взаимодействие с персонажем-ёжиком.

### 1.2. Цели и задачи
- Создать увлекательную игровую среду для обучения основам счета
- Обеспечить адаптивную систему сложности для соответствия различным возрастным группам
- Разработать интуитивно понятный интерфейс для детей
- Включить элементы поощрения и положительного подкрепления
- Обеспечить автономную работу игры без необходимости подключения к интернету

### 1.3. Целевая аудитория
- Дети от 3 до 7 лет, изучающие основы счета
- Родители и педагоги, занимающиеся с детьми

## 2. Технические требования

### 2.1. Платформы
- Веб-браузеры (Chrome, Firefox, Safari, Edge)
- Поддержка настольных и мобильных устройств
- Автономная работа без сервера и интернет-соединения

### 2.2. Технические спецификации
- Используемые технологии: HTML5, CSS3, JavaScript
- Минимальное разрешение экрана: 768x576
- Поддержка сенсорных экранов
- Минимальные требования к устройству: поддержка HTML5 и аудио API

### 2.3. Ограничения
- Игра должна работать автономно без необходимости установки дополнительного ПО
- Оптимизация для низкопроизводительных устройств (ограничение анимаций)
- Объем игры не должен превышать 10 МБ

## 3. Функциональные требования

### 3.1. Основной игровой процесс
1. **Главный экран**
   - Название игры
   - Кнопка "Начать игру"
   - Простая инструкция

2. **Игровой экран**
   - Интерактивное дерево с яблоками
   - Персонаж-ёжик
   - Счетчик собранных яблок
   - Кнопка включения/выключения звука

3. **Игровой процесс**
   - При нажатии на яблоко, оно падает и ёжик его ловит
   - После сбора всех яблок, игрок должен выбрать число, соответствующее количеству собранных яблок
   - При правильном ответе появляется экран поздравления
   - При неправильном ответе игрок получает возможность повторить попытку
   - После 2-х неправильных попыток игрок получает подсказку

### 3.2. Система сложности
- Начальный уровень: 1 яблоко
- Увеличение сложности после 3-х правильных ответов подряд
- Максимальное количество яблок: 10
- Адаптивные подсказки в зависимости от возраста и сложности уровня

### 3.3. Аудио
- Фоновая музыка (с возможностью отключения)
- Звуковые эффекты при сборе яблок
- Звуковые эффекты при правильном и неправильном ответах
- Голосовые подсказки на русском языке

### 3.4. Визуальный стиль
- Яркие, детские цвета
- Анимированные персонажи
- Крупные интерактивные элементы для удобства нажатия
- Понятные визуальные индикаторы прогресса

## 4. Игровые элементы

### 4.1. Персонажи
1. **Ёжик**
   - Анимированный персонаж с реакциями на действия игрока
   - Анимации: ожидание, движение, поедание яблока, радость

2. **Яблоки**
   - Интерактивные объекты, реагирующие на нажатие
   - Анимация падения с дерева
   - Визуальное отличие собранных и несобранных яблок

3. **Дерево**
   - Статичный фон с интерактивными элементами (яблоками)
   - Визуально привлекательный дизайн

### 4.2. Интерфейс
1. **Кнопки выбора числа**
   - Крупные, цветные кнопки с цифрами
   - Визуальная реакция при нажатии
   - Логическое расположение (по порядку или в круге)

2. **Счетчики и индикаторы**
   - Счетчик текущего количества собранных яблок
   - Индикация текущего уровня сложности
   - Индикация правильных/неправильных ответов

3. **Экраны состояния**
   - Экран успеха с поздравлением
   - Экран выбора числа
   - Экран подсказки при неправильных ответах

## 5. Пользовательские сценарии

### 5.1. Базовый сценарий игры
1. Игрок открывает главный экран и нажимает "Начать игру"
2. Появляется игровой экран с деревом и ёжиком
3. Игрок нажимает на яблоки, и ёжик их собирает
4. После сбора всех яблок, игрок переходит к экрану выбора числа
5. Игрок выбирает число, соответствующее количеству собранных яблок
6. При правильном ответе игрок получает поздравление и переходит к следующему уровню
7. При неправильном ответе игрок получает возможность попробовать еще раз

### 5.2. Дополнительные сценарии
1. **Сценарий с подсказкой**
   - Игрок дважды выбирает неправильное число
   - Появляется подсказка с правильным ответом
   - Игрок выбирает правильное число и продолжает игру

2. **Сценарий повышения сложности**
   - Игрок правильно отвечает на 3 вопроса подряд
   - Сложность увеличивается (добавляется еще одно яблоко)
   - Игрок продолжает игру на новом уровне сложности

2. **Результаты**

### 7.2. Голосовое сопровождение

3. **Поздравления**

## 11. Детализация поэтапного отображения игровых объектов

### 11.1. Главный экран (Начальный этап)

**Объекты для отображения:**
- Фоновое изображение с природным мотивом (светлые тона)
- Название игры "Яблоки и ёжик" (крупный шрифт с тенью)
- Статичное изображение ёжика (приветственная поза)
- Кнопка "Начать игру" (яркий зеленый цвет, анимация пульсации)
- Кнопка включения/выключения звука (правый верхний угол)
- Декоративные элементы: 2-3 яблока в верхней части экрана

**Порядок появления:**
1. Фоновое изображение (моментально)
2. Название игры (плавное появление, 0.5 сек)
3. Изображение ёжика (появление с небольшим подпрыгиванием, 0.7 сек)
4. Кнопка "Начать игру" (проявление с эффектом пульсации, 1 сек)
5. Декоративные элементы (плавное проявление, 1.2 сек)

### 11.2. Игровой экран: фаза сбора яблок

**Объекты для отображения:**
- Фон игровой сцены (светло-зеленый с текстурой травы)
- Яблочное дерево (центральное изображение)
- Яблоки на дереве (количество зависит от текущего уровня сложности)
- Ёжик (анимированный персонаж внизу экрана)
- Счетчик собранных яблок (правый верхний угол)
- Кнопка звука (правый верхний угол)
- Номер текущего уровня (левый верхний угол)

**Порядок появления:**
1. Фон игровой сцены (моментально)
2. Яблочное дерево (плавное проявление, 0.8 сек)
3. Ёжик (появление снизу экрана, 1 сек)
4. Яблоки на дереве (последовательное появление с интервалом 0.2 сек)
5. Интерфейсные элементы: счетчик, кнопка звука, номер уровня (одновременное проявление, 1.5 сек)
6. Всплывающая подсказка "Собери яблоки с дерева!" (появляется на 3 секунды и плавно исчезает)

**Особенности интерактивности:**
- Яблоки: при наведении курсора слегка увеличиваются и покачиваются
- Яблоки при нажатии: анимация отрыва от ветки и падения
- Ёжик: автоматически движется к месту падения яблока
- Счетчик: увеличивается с анимацией при каждом собранном яблоке

### 11.3. Переходный этап: от сбора яблок к выбору числа

**Условие перехода:**
Все яблоки текущего уровня собраны.

**Объекты для отображения во время перехода:**
- Затемнение игрового экрана (полупрозрачный черный цвет, 50% непрозрачности)
- Анимация перехода (вращающееся яблоко в центре экрана)
- Текст "Молодец!" (появление и исчезновение)

**Порядок анимации:**
1. Затемнение фона (плавное, 0.5 сек)
2. Вращающееся яблоко (появление с увеличением, 0.8 сек)
3. Текст "Молодец!" (проявление, задержка 1 сек, исчезновение)
4. Плавный переход к экрану выбора числа (смена экрана слева направо, 1 сек)

### 11.4. Игровой экран: фаза выбора числа

**Объекты для отображения:**
- Фон (более светлый, чем в фазе сбора яблок)
- Визуализация собранных яблок (центральное изображение)
- Текст вопроса "Сколько яблок собрал ёжик?" (вверху)
- Кнопки с числами для выбора (расположены по горизонтали или в круге)
- Мини-изображение ёжика (в ожидательной позе)
- Кнопка звука (правый верхний угол)

**Порядок появления:**
1. Фон (моментально)
2. Текст вопроса (плавное проявление сверху, 0.7 сек)
3. Визуализация собранных яблок (появление с небольшой анимацией, 1 сек)
4. Кнопки с числами (последовательное появление слева направо, 1.3 сек)
5. Мини-изображение ёжика (проявление с анимацией, 1.5 сек)

**Варианты чисел:**
- Всегда включать правильный ответ (количество собранных яблок)
- 1-2 числа рядом с правильным ответом (±1 или ±2)
- 1 число, значительно отличающееся от правильного ответа

**Поведение при неправильном ответе:**
1. Выбранная кнопка подсвечивается красным
2. Анимация встряхивания кнопки
3. После 2-х неверных попыток:
   - Отображение подсказки над кнопками
   - Мигающая подсветка правильной кнопки
   - Затемнение неправильных кнопок

### 11.5. Экран успеха (при правильном ответе)

**Объекты для отображения:**
- Фон "Праздничный" с элементами конфетти
- Крупная надпись "Молодец!"
- Изображение ёжика (в радостной позе)
- Визуализация правильного ответа (цифра и соответствующее количество яблок)
- Текст похвалы "Ты правильно посчитал, сколько яблок собрал ёжик!"
- Кнопка "Продолжить" (для перехода к следующему уровню)

**Порядок появления:**
1. Праздничный фон с конфетти (анимация выстрела конфетти, 0.5 сек)
2. Надпись "Молодец!" (появление с увеличением, 0.7 сек)
3. Изображение радостного ёжика (появление с подпрыгиванием, 0.9 сек)
4. Визуализация правильного ответа (плавное проявление, 1.1 сек)
5. Текст похвалы (постепенное появление по словам, 1.5 сек)
6. Кнопка "Продолжить" (проявление с пульсацией, 2 сек)

**Действия при переходе:**
- Анимация затухания экрана успеха
- Подготовка нового уровня (генерация новой конфигурации яблок)
- Увеличение уровня сложности, если выполнены условия (3 правильных ответа подряд)

### 11.6. Экран подсказки (после 2-х неверных попыток)

**Объекты для отображения:**
- Полупрозрачное наложение на экран выбора числа
- Текст подсказки "Ёжик собрал [правильное число] яблок. Давай попробуем ещё раз!"
- Визуальное выделение правильного числа
- Анимированная стрелка, указывающая на правильный ответ

**Порядок появления:**
1. Полупрозрачное наложение (плавное проявление, 0.5 сек)
2. Текст подсказки (появление с эффектом печатающегося текста, 1 сек)
3. Выделение правильного числа (пульсирующее свечение, 1.3 сек)
4. Анимированная стрелка (появление с движением, 1.5 сек)

**Действия после подсказки:**
- Ожидание взаимодействия игрока
- При нажатии на правильное число: переход к экрану успеха
- Счетчик правильных ответов подряд обнуляется

### 11.7. Экран повышения сложности (после 3-х правильных ответов подряд)

**Объекты для отображения:**
- Переходный экран с градиентным фоном
- Текст "Отлично! Переходим на следующий уровень"
- Анимация увеличения числа (старый уровень → новый уровень)
- Изображение ёжика в "боевой" позе
- Дополнительное яблоко с эффектом сияния

**Порядок появления:**
1. Градиентный фон (плавное проявление, 0.5 сек)
2. Текст о переходе на новый уровень (появление сверху, 0.8 сек)
3. Анимация изменения числа уровня (1 сек)
4. Изображение ёжика (появление слева, 1.2 сек)
5. Дополнительное яблоко с сиянием (появление справа, 1.5 сек)
6. Автоматический переход к экрану сбора яблок через 3 секунды

## 12. Специальные состояния объектов

### 12.1. Состояния яблок
1. **Статическое состояние** (на дереве)
   - Небольшая анимация покачивания
   - При наведении: увеличение и изменение тени
   
2. **Падающее состояние**
   - Анимация отрыва от ветки
   - Анимация вращения при падении
   - Физика падения с ускорением
   
3. **Собранное состояние**
   - Краткое отображение на спине ёжика
   - Плавное исчезновение с уменьшением размера
   - Обновление счетчика собранных яблок

### 12.2. Состояния ёжика
1. **Ожидание**
   - Небольшая анимация дыхания
   - Периодическое моргание глаз
   - Случайные небольшие движения

2. **Движение к яблоку**
   - Анимация бега (ускорение в начале, замедление при приближении)
   - Направление движения зависит от позиции падающего яблока
   - Скорость движения пропорциональна расстоянию

3. **Поедание яблока**
   - Анимация подпрыгивания
   - Изменение выражения "лица" (радость)
   - Звуковой эффект поедания

4. **Реакция на правильный ответ**
   - Прыжки на месте
   - Вращение или танец
   - Выражение радости

5. **Реакция на неправильный ответ**
   - Покачивание головой
   - Грустное выражение "лица"
   - Утешительный жест

### 12.3. Состояния кнопок с числами
1. **Базовое состояние**
   - Цветовое оформление в зависимости от числа
   - Небольшая тень для трехмерного эффекта
   
2. **При наведении/фокусе**
   - Увеличение размера (105-110%)
   - Усиление яркости
   - Увеличение тени
   
3. **При нажатии на правильный ответ**
   - Зеленая подсветка
   - Анимация пульсации
   - Звуковой эффект успеха
   
4. **При нажатии на неправильный ответ**
   - Красная подсветка
   - Анимация встряхивания
   - Звуковой эффект ошибки
   
5. **В режиме подсказки**
   - Правильная кнопка: яркое свечение
   - Неправильные кнопки: затемнение (50% непрозрачность)

## 13. Точная последовательность загрузки и отображения ресурсов

### 13.1. Предзагрузка ресурсов
1. **При запуске игры**
   - Основные изображения (фоны, ёжик, дерево)
   - Критичные звуковые эффекты (успех, ошибка)
   - Базовые элементы интерфейса

2. **Отложенная загрузка во время начального экрана**
   - Дополнительные анимации
   - Дополнительные звуковые эффекты
   - Ресурсы для последующих уровней

### 13.2. Экран загрузки
1. **Элементы на экране загрузки**
   - Логотип игры
   - Изображение ёжика в позе ожидания
   - Полоса загрузки (отображает реальный прогресс загрузки)
   - Процент загрузки (числовое отображение)

2. **Последовательность появления**
   - Фон (моментально)
   - Логотип (плавное проявление, 0.5 сек)
   - Изображение ёжика (появление с анимацией, 0.8 сек)
   - Полоса загрузки (появление, 1 сек)
   - Обновление прогресса в реальном времени
   - Переход к главному экрану по завершении (плавное затухание, 1 сек)

### 13.3. Последовательность реакций на действия пользователя

1. **При клике на яблоко**
   1. Визуальная реакция яблока (0.1 сек)
   2. Начало анимации падения (0.2 сек)
   3. Начало движения ёжика к точке падения (0.3 сек)
   4. Падение яблока и "ловля" ёжиком (0.8-1.2 сек)
   5. Анимация поедания (1.3 сек)
   6. Обновление счетчика яблок (1.5 сек)
   7. Звуковые эффекты синхронизированы с визуальными событиями

2. **При выборе числа**
   1. Визуальная реакция кнопки (0.1 сек)
   2. Звуковой эффект нажатия (одновременно)
   3. При правильном ответе:
      - Зеленая подсветка кнопки (0.2 сек)
      - Звук успеха (0.3 сек)
      - Анимация ёжика (радость) (0.4 сек)
      - Начало перехода к экрану успеха (1 сек)
   4. При неправильном ответе:
      - Красная подсветка кнопки (0.2 сек)
      - Звук ошибки (0.3 сек)
      - Анимация ёжика (грусть) (0.4 сек)
      - Анимация встряхивания кнопки (0.5 сек)
      - Возврат к выбору (кнопка становится неактивной) (0.8 сек)

3. **При достижении порога повышения сложности**
   1. Завершение экрана успеха (нормальная анимация)
   2. Отображение экрана повышения уровня (вместо обычного перехода)
   3. Специальный звуковой эффект повышения уровня
   4. Визуальные спецэффекты (сияние, конфетти)
   5. Переход к новой конфигурации игры через 3 секунды

## 14. Точки принятия решений и ветвления игрового процесса

### 14.1. Принятие решений системой
1. **После сбора всех яблок**
   - Принятие решения о переходе к фазе выбора числа
   - Генерация вариантов ответов (включая правильный)
   - Определение размещения вариантов на экране

2. **После ответа игрока**
   - Проверка правильности ответа
   - Счетчик неправильных попыток
   - Счетчик правильных ответов подряд
   - Оценка необходимости подсказки или повышения сложности

3. **После 3-х правильных ответов подряд**
   - Проверка текущего уровня сложности
   - Если < 10, генерация экрана повышения сложности
   - Если = 10, стандартный переход к следующему уровню без повышения

### 14.2. Критические моменты взаимодействия
1. **Порядок генерации объектов при начале уровня**
   - Сначала фон и статические элементы
   - Затем дерево без яблок
   - Затем ёжик
   - В последнюю очередь яблоки (с анимацией появления)

2. **Синхронизация действий при сборе яблок**
   - Расчет траектории падения яблока
   - Расчет точки прибытия ёжика
   - Синхронизация прибытия ёжика и яблока в одну точку
   - Проверка условия перехода (все яблоки собраны?)

3. **Процедура завершения уровня**
   - Сохранение счетчика правильных ответов подряд
   - Обновление уровня сложности (если нужно)
   - Подготовка новой конфигурации яблок
   - Сброс счетчиков текущего уровня

## 15. Основной алгоритм игры на языке BASIC

```basic
REM =============================================
REM ИГРА "ЯБЛОКИ И ЁЖИК" - ОСНОВНОЙ АЛГОРИТМ
REM Версия: 1.0
REM =============================================

REM Инициализация основных переменных
LET GAME_STATE = "MAIN_MENU"       'Текущее состояние игры
LET MAX_APPLES = 1                 'Максимальное количество яблок на текущем уровне
LET APPLES_COLLECTED = 0           'Счетчик собранных яблок
LET INCORRECT_ATTEMPTS = 0         'Счетчик неправильных попыток
LET CORRECT_STREAK = 0             'Счетчик правильных ответов подряд
LET SOUND_ENABLED = 1              'Флаг включения звука (1=ВКЛ, 0=ВЫКЛ)

REM Инициализация массива для хранения координат яблок
DIM APPLE_X(10)                    'X-координаты яблок
DIM APPLE_Y(10)                    'Y-координаты яблок
DIM APPLE_COLLECTED(10)            'Статус сбора (0=на дереве, 1=собрано)

REM Инициализация координат ёжика
LET HEDGEHOG_X = 400               'Начальная X-координата ёжика
LET HEDGEHOG_Y = 500               'Начальная Y-координата ёжика
LET HEDGEHOG_STATE = "IDLE"        'Начальное состояние ёжика

REM =============================================
REM ОСНОВНОЙ ИГРОВОЙ ЦИКЛ
REM =============================================
MAIN_LOOP:
  REM Проверка текущего состояния игры
  IF GAME_STATE = "MAIN_MENU" THEN GOSUB DRAW_MAIN_MENU
  IF GAME_STATE = "COLLECTING" THEN GOSUB DRAW_COLLECTION_PHASE
  IF GAME_STATE = "SELECTING" THEN GOSUB DRAW_SELECTION_PHASE
  IF GAME_STATE = "SUCCESS" THEN GOSUB DRAW_SUCCESS_SCREEN
  IF GAME_STATE = "HINT" THEN GOSUB DRAW_HINT_SCREEN
  IF GAME_STATE = "LEVEL_UP" THEN GOSUB DRAW_LEVEL_UP_SCREEN

  REM Обработка ввода и событий
  GOSUB HANDLE_EVENTS

  REM Обновление анимаций
  GOSUB UPDATE_ANIMATIONS
  
  REM Задержка для контроля FPS
  SLEEP 16  'примерно 60 FPS
  
  REM Повторение цикла
  GOTO MAIN_LOOP

REM =============================================
REM ПОДПРОГРАММЫ ОТРИСОВКИ ЭКРАНОВ
REM =============================================

REM Отрисовка главного меню
DRAW_MAIN_MENU:
  CLS  'Очистка экрана
  PRINT "ЯБЛОКИ И ЁЖИК"
  PRINT "Нажмите ПРОБЕЛ, чтобы начать игру"
  
  IF SOUND_ENABLED = 1 THEN
    PRINT "Звук: ВКЛ (нажмите S для переключения)"
  ELSE
    PRINT "Звук: ВЫКЛ (нажмите S для переключения)"
  END IF
  
  RETURN

REM Отрисовка фазы сбора яблок
DRAW_COLLECTION_PHASE:
  CLS  'Очистка экрана
  
  REM Отрисовка дерева (фоновое изображение)
  GOSUB DRAW_TREE
  
  REM Отрисовка ёжика
  GOSUB DRAW_HEDGEHOG
  
  REM Отрисовка яблок на дереве
  FOR I = 1 TO MAX_APPLES
    IF APPLE_COLLECTED(I) = 0 THEN
      GOSUB DRAW_APPLE(I)
    END IF
  NEXT I
  
  REM Отрисовка счетчика собранных яблок
  PRINT "Собрано яблок: ";APPLES_COLLECTED
  
  REM Проверка, все ли яблоки собраны
  IF APPLES_COLLECTED = MAX_APPLES THEN
    REM Если все яблоки собраны, переходим к выбору числа
    GAME_STATE = "SELECTING"
    SLEEP 1000  'Задержка перед переходом
  END IF
  
  RETURN

REM Отрисовка фазы выбора числа
DRAW_SELECTION_PHASE:
  CLS  'Очистка экрана
  
  REM Отображение вопроса
  PRINT "Сколько яблок собрал ёжик?"
  
  REM Отображение собранных яблок
  GOSUB DRAW_COLLECTED_APPLES
  
  REM Генерация и отображение вариантов ответа
  GOSUB GENERATE_NUMBER_OPTIONS
  GOSUB DRAW_NUMBER_OPTIONS
  
  RETURN

REM Отрисовка экрана успеха
DRAW_SUCCESS_SCREEN:
  CLS  'Очистка экрана
  PRINT "МОЛОДЕЦ!"
  PRINT "Ты правильно посчитал, сколько яблок собрал ёжик!"
  PRINT "Яблок: ";APPLES_COLLECTED
  
  REM Отображение собранных яблок
  GOSUB DRAW_COLLECTED_APPLES
  
  REM Отображение радостного ёжика
  GOSUB DRAW_HAPPY_HEDGEHOG
  
  PRINT "Нажмите ПРОБЕЛ для продолжения"
  
  RETURN
```
